array-type

规定数组使用 T[] 或者 Array<T>。

在你的代码库中，使用相同的数组定义风格能够使得开发人员更容易阅读和理解类型。

Rule Details​

这条规则宗旨是在你的代码库中标准化数组类型。

Options​

type ArrayOption = "array" | "generic" | "array-simple";
type Options = {
  default: ArrayOption;
  readonly?: ArrayOption;
};

const defaultOptions: Options = {
  default: "array",
};

这条规则接受具有以下属性的 options 对象：

  • default - 设置数组类型预期是可变的。
  • readonly - 设置数组类型预期是只读数组。如果省略，则将使用default。

每个属性都可以设置为这三个中的一个： 'array' | 'generic' | 'array-simple'。

默认配置将强制所有的可变的和只读的数组使用 array 语法。

"array"​

对于所有的数组类型，始终使用 T[] 或者 readonly T[]。

❌ Incorrect​

const x: Array<string> = ["a", "b"];
const y: ReadonlyArray<string> = ["a", "b"];

✅ Correct​

const x: string[] = ["a", "b"];
const y: readonly string[] = ["a", "b"];

"generic"​

对于所有的数组类型，始终使用 Array<T> 或者 ReadonlyArray<T>。

❌ Incorrect​

const x: string[] = ["a", "b"];
const y: readonly string[] = ["a", "b"];

✅ Correct​

const x: Array<string> = ["a", "b"];
const y: ReadonlyArray<string> = ["a", "b"];

"array-simple"​

对于简单类型(类型仅是原始名称或者类型引用)使用 T[] 或者 readonly T[]。 对于其他类型(联合类型、交叉类型、对象类型、函数类型)使用 Array<T> 或者 ReadonlyArray<T>。

❌ Incorrect​

const a: (string | number)[] = ["a", "b"];
const b: { prop: string }[] = [{ prop: "a" }];
const c: (() => void)[] = [() => {}];
const d: Array<MyType> = ["a", "b"];
const e: Array<string> = ["a", "b"];
const f: ReadonlyArray<string> = ["a", "b"];

✅ Correct​

const a: Array<string | number> = ["a", "b"];
const b: Array<{ prop: string }> = [{ prop: "a" }];
const c: Array<() => void> = [() => {}];
const d: MyType[] = ["a", "b"];
const e: string[] = ["a", "b"];
const f: readonly string[] = ["a", "b"];
